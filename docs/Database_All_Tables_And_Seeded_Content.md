# WorkAudit Oracle Database Dump: Tables and Seed Content

This file is a single source of truth summary for all tables in WorkAudit and the seed/default rows that are inserted by the application at first run and migration startup.

## Scope

- Provider: `Oracle.ManagedDataAccess.Client`
- Baseline installer version: `49`
- Migration range currently included: `50` through `55`
- This file is generated from source-of-truth schema/migration/seed code in the repository.

## Migration-level changes applied after baseline

- `50 - Normalize required document text columns`
  - `documents.source` defaulted to `Unknown` when null/blank.
  - `documents.engagement` defaulted to `General` when null/blank.
  - `documents.section` defaulted to `Unspecified` when null/blank.
- `51 - Normalize event time columns`
  - Renames `timestamp` columns to `event_time` for:
    - `audit_log`
    - `report_distributions`
  - Converts event columns to `TIMESTAMP(6)`.
  - Rebuilds timestamp-oriented indexes to use `event_time`.
- `52 - Oracle backup app settings`
  - Adds app_settings keys for Data Pump backup:
    - `include_oracle_data`
    - `oracle_datapump_directory`
    - `oracle_datapump_local_folder`
    - `oracle_backup_dump_tool_path`
    - `oracle_backup_retention_days`
- `53 - Scheduler leader election`
  - Adds table `workaudit_scheduler_locks`.
  - Adds app_settings keys:
    - `scheduler_leader_election_enabled`
    - `scheduler_lock_lease_minutes`
    - `scheduled_report_last_run_date`
- `54 - Remove obsolete AI settings`
  - Removes legacy AI keys from `app_settings` when present:
    - `classification_confidence_threshold`
    - `vision_extraction_enabled`
    - `vision_model_name`
    - `vision_timeout_seconds`
    - `ollama_model`
    - `ollama_endpoint`
  - Deletes any rows with `category = 'ai'`.
- `55 - Cleanup obsolete AI app settings (case-insensitive)`
  - Re-runs the same legacy key deletions and removes any `app_settings` rows whose trimmed `category` matches `ai` ignoring case (covers rows migration 54 missed).

## Live database dump script (PowerShell)

Create and run this script to generate a fresh `.md` with full row content for each table.

```powershell
param(
  [string]$ConnectionString = "User Id=WORKAUDIT;Password=YOUR_PASSWORD;Data Source=//localhost:1521/XEPDB1",
  [string]$OutputFile = "g:\\WorkAudit.CSharpOracle\\docs\\database_live_content.md"
)

Add-Type -Path (Get-ChildItem -Path "g:\\WorkAudit.CSharpOracle" -Recurse -Filter "Oracle.ManagedDataAccess.dll" | Select-Object -First 1).FullName

$conn = New-Object Oracle.ManagedDataAccess.Client.OracleConnection($ConnectionString)
$conn.Open()

$sb = New-Object System.Text.StringBuilder
$append = { param($line) [void]$sb.AppendLine($line) }

&$append "# Database Live Content Export"
&$append ""
&$append "Generated: $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"
&$append ""

$tableCmd = $conn.CreateCommand()
$tableCmd.CommandText = "SELECT table_name FROM user_tables WHERE table_name NOT LIKE 'BIN$%' ORDER BY table_name"
$tables = New-Object System.Collections.Generic.List[string]
$r = $tableCmd.ExecuteReader()
while ($r.Read()) { $tables.Add($r.GetString(0)) }
$r.Close()

foreach ($table in $tables) {
  &$append "## $table"

  $countCmd = $conn.CreateCommand()
  $countCmd.CommandText = "SELECT COUNT(*) FROM ""$table"""
  $count = [int]$countCmd.ExecuteScalar()
  &$append "Rows: $count"

  $colCmd = $conn.CreateCommand()
  $colCmd.CommandText = "SELECT column_name FROM user_tab_columns WHERE table_name = :t ORDER BY column_id"
  [void]$colCmd.Parameters.Add("t", [Oracle.ManagedDataAccess.Client.OracleDbType]::Varchar2, $table, [System.Data.ParameterDirection]::Input)

  $cols = @()
  $colR = $colCmd.ExecuteReader()
  while ($colR.Read()) { $cols += $colR.GetString(0) }
  $colR.Close()

  $ddl = "SELECT dbms_metadata.get_ddl('TABLE', :t) FROM dual"
  $ddlCmd = $conn.CreateCommand()
  $ddlCmd.CommandText = $ddl
